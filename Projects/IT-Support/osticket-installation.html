Prerequisites & System Requirements

Before beginning the osTicket installation, ensure your system meets all required specifications. This section outlines what you'll need.

1

System Requirements

Verify your server meets minimum specifications

Hardware Requirements

  • Processor: 1 GHz or faster CPU (2+ cores recommended)
  • RAM: Minimum 512 MB (2 GB recommended for production)
  • Disk Space: At least 500 MB for installation
  • Network: Active internet connection for package downloads

Software Requirements

  • Operating System: Linux (Ubuntu 20.04 LTS recommended)
  • Web Server: Apache 2.4.x with mod_rewrite enabled
  • PHP: 8.0+ (8.3+ recommended) with required extensions
  • Database: MySQL 5.7+ or MariaDB 10.0+
Tip: For this guide, we're using Ubuntu 20.04 LTS with Apache, PHP, and MySQL. All commands are specific to this setup.

Terminal output confirming system meets requirements.

Installing Required Dependencies

Update your system and install all required packages. This includes PHP modules, Apache components, and database tools.

2

System Update & Initial Packages

Update system and install foundational packages

Update Package Manager

First, update your package manager:

sudo apt update && sudo apt upgrade -y

Install Apache and PHP

Install Apache web server with PHP

sudo apt install apache2 php php-mysql libapache2-mod-php

Screenshot: Package Installation in Progress

Show terminal during apt install output.

3

Install MySQL

Install required database server

Install Mariadb Server

Install Mariadb 8.0:

sudo apt install -y mariadb-server
Important: After installing MariaDB, run sudo mariadb_secure_installation to secure your database.

Screenshot: MySQL Installation Complete

Show completion message confirming all packages installed successfully.

Configure Apache Web Server

Create an Apache virtual host for osTicket and enable required modules for proper functionality.

4

Create Virtual Host & Enable Modules

Set up Apache configuration for osTicket

Create Virtual Host Configuration

Create a new Apache configuration file:

sudo nano /etc/apache2/sites-available/osticket.conf

Enable Modules and Virtual Host

Enable mod_rewrite and activate the site:

sudo a2enmod rewrite
sudo a2ensite osticket.conf
sudo apache2ctl configtest
sudo systemctl restart apache2

Screenshot: Apache Configuration Complete

Show terminal with "Syntax OK" message and modules enabled.

Setup MySQL Database

Create a dedicated MySQL database and user account for osTicket with appropriate privileges.

5

Create Database & User Account

Set up MySQL authentication and database

Access MySQL and Execute Commands

Log in as root and create the database:

sudo mysql -u root -p

CREATE DATABASE osticket;
CREATE USER 'osticket'@'localhost' IDENTIFIED BY 'securepassword123';
GRANT ALL PRIVILEGES ON osticket.* TO 'osticket'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Security: Use a strong, unique password instead of 'securepassword123'. Store it securely for the installation wizard.

Screenshot: MySQL Database Created

Show MySQL prompt with database and user creation commands executed.

Deploy osTicket Application

Download the osTicket application, extract it to the web root, and configure file permissions.

6

Download & Extract osTicket

Deploy application files and set permissions

Download osTicket Release

Download and extract the latest stable version:

cd /var/www/html/
sudo wget "https://s3.amazonaws.com/...
sudo unzip osTicket-v1.16.0.zip
sudo rm osTicket-v1.16.0.zip

Set File Permissions & Ownership

Configure proper permissions for the web server:

sudo chown -R www-data:www-data /var/www/html/osTicket
sudo chmod -R 755 /var/www/html/osTicket/upload
sudo chmod 777 /var/www/osticket/upload/include

Verify Installation & Secure

Complete final security hardening and verify that osTicket is fully functional.

8

Post-Installation Security Hardening

Remove installation files and lock down permissions

Remove Setup Directory

Delete the installer to prevent unauthorized access:

sudo rm -rf /var/www/html/osTicket/setup

Restrict File Permissions

Lock down the configuration directory:

sudo chmod 755 /var/www/html/osTicket/include

Verify Installation Complete

Test both admin and client interfaces

http://localhost/osTicket/admin

Test the client-facing ticket submission interface:
