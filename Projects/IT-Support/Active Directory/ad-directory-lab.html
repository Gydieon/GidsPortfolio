Create Organizational Units (OUs)

Organizational Units (OUs) are the building blocks of an Active Directory structure, allowing for logical grouping of users, computers, and other resources. In this section, we will create OUs to represent different departments within the company, such as "HR," "IT," and "Management." This hierarchical structure helps in applying group policies and managing permissions effectively across the organization.

1

Create OUs

Organizational Units (OUs) are the building blocks of an Active Directory structure.

Steps to Create OUs

  • Open Active Directory Users and Computers
  • Right-click the domain: lab.local
  • Select New → Organizational Unit
  • Created the following OUs: HR, IT, Management

Create OUs

The Departments (OUs) are created

Why Creating Organizational Units (OUs) is Important

In Active Directory, Organizational Units (OUs) are containers used to logically organize users, computers, and other directory objects within a domain. Creating OUs is essential for structuring the environment in a way that reflects the organization’s departments, roles, or locations. This logical separation allows administrators to efficiently manage resources, delegate administrative control, and apply Group Policies in a targeted manner. Proper OU design improves scalability, security, and overall management of the Active Directory environment.

  • OUs act as containers for organizing users, computers, and other directory objects
  • Reflects the organizational structure such as departments (HR, IT, Management)
  • Allows targeted application of Group Policy Objects (GPOs)
  • Enables delegation of administrative control without giving full domain access
  • Improves management and troubleshooting by grouping related objects together
  • Supports scalability as the organization grows and adds more users and resources

Create Users

This stage focuses on creating user accounts within Active Directory to represent employees across different departments. By organizing users into their respective Organizational Units (OUs), we establish a structured and scalable directory that reflects a real-world enterprise environment and supports efficient identity and access management.

2

Create User Accounts

Organize user accounts within their respective OUs for effective management.

Why User Creation is Important

In Active Directory, users are treated as objects within the directory, meaning each user account is a stored entity with defined attributes such as username, password, email, and group membership. Creating user accounts is a fundamental part of Active Directory management, as it enables centralized authentication, authorization, and control of access to resources across the organization. Properly structured user objects ensure that access can be efficiently managed, monitored, and scaled in a real-world enterprise environment.

  • Users are directory objects with attributes used for identification and access control
  • Represents employees within the organization for centralized identity management
  • Allows users to authenticate and access domain resources securely
  • Supports role-based access control when combined with security groups
  • Improves organization by placing users into department-specific OUs
  • Prepares the environment for applying Group Policies and permissions

Steps to Create Users in OUs

  • Open Active Directory Users and Computers
  • Navigate to the desired Organizational Unit (e.g., HR, IT, Management)
  • Right-click the OU → New → User
  • Enter user details (username and full name)
  • Set a password for the user account
  • Uncheck "User must change password at next login" (lab setup)
  • Complete the wizard to create the user
  • Repeat the process for each department OU

Create Security Groups

Security groups are essential for managing permissions and access control within an Active Directory environment. By creating security groups and assigning users to them, administrators can efficiently control access to shared resources such as files, folders, and applications. This method simplifies management and improves security by assigning permissions to groups rather than individual users.

4

Creating Security Groups in Active Directory

Follow these steps to create and organize security groups:

Steps to Create Security Groups

  • Open Active Directory Users and Computers
  • Navigate to the appropriate Organizational Unit (e.g., HR, IT, Management)
  • Right-click inside the OU and select New → Group
  • Enter the group name (e.g., adminFInance, adminIT)
  • Set Group scope to Global
  • Set Group type to Security
  • Click OK to create the group

Screenshot: Created Security Groups

The image below shows successfully created one security group within its respective Organizational Units. This group will be used to manage user permissions and access to network resources.

Why Security Groups are Important

  • Simplifies permission management by assigning access to groups instead of individual users
  • Enhances security through centralized access control
  • Reduces administrative workload in large environments
  • Makes it easier to onboard or remove users by adding/removing them from groups
  • Follows industry best practices used in enterprise IT environments
  • Improves organization and scalability as the network grows
