Prerequisites & System Requirements

Before beginning the osTicket installation, ensure your system meets all required specifications. This section outlines what you'll need.

1

System Requirements

Verify your server meets minimum specifications

Hardware Requirements

  • Processor: 1 GHz or faster CPU (2+ cores recommended)
  • RAM: Minimum 512 MB (2 GB recommended for production)
  • Disk Space: At least 500 MB for installation
  • Network: Active internet connection for package downloads

Software Requirements

  • Operating System: Linux (Ubuntu 20.04 LTS recommended)
  • Web Server: Apache 2.4.x with mod_rewrite enabled
  • PHP: 7.2+ (7.4+ recommended) with required extensions
  • Database: MySQL 5.7+ or MariaDB 10.0+
Tip: For this guide, we're using Ubuntu 20.04 LTS with Apache, PHP 7.4, and MySQL 8.0. All commands are specific to this setup.

Installing Required Dependencies

Update your system and install all required packages. This includes PHP modules, Apache components, and database tools.

2

System Update & Initial Packages

Update system and install foundational packages

Update Package Manager

First, update your package manager to fetch the latest package lists:

sudo apt update && sudo apt upgrade -y

This ensures you have the latest security patches and package information.

Install Apache and PHP

Install Apache web server with PHP 7.4 support:

sudo apt install -y apache2 php7.4 libapache2-mod-php7.4

Install Required PHP Modules

osTicket requires several PHP extensions. Install them all at once:

sudo apt install -y php7.4-mysql php7.4-curl php7.4-gd php7.4-imap php7.4-xml php7.4-mbstring php7.4-intl php7.4-apcu

Install MySQL Server

Install MySQL 8.0 for the osTicket database:

sudo apt install -y mysql-server
Important: After installing MySQL, run sudo mysql_secure_installation to secure your database instance. Set a strong root password.
3

Enable Required Apache Modules

Enable mod_rewrite and set correct permissions

Enable Apache Modules

osTicket requires mod_rewrite for URL rewriting:

sudo a2enmod rewrite
sudo a2enmod php7.4

After enabling modules, restart Apache:

sudo systemctl restart apache2

Configure Apache Web Server

Create an Apache virtual host for osTicket and configure rewrite rules for clean URLs.

4

Create Virtual Host

Set up Apache configuration for osTicket

Create Configuration File

Create a new Apache virtual host configuration:

sudo nano /etc/apache2/sites-available/osticket.conf

Paste the following configuration:

<VirtualHost *:80>
ServerName osticket.local
ServerAdmin admin@osticket.local
DocumentRoot /var/www/osticket/upload

<Directory /var/www/osticket/upload>
AllowOverride All
Require all granted
</Directory>

ErrorLog ${APACHE_LOG_DIR}/osticket-error.log
CustomLog ${APACHE_LOG_DIR}/osticket-access.log combined
</VirtualHost>

Enable Virtual Host

Enable the osTicket virtual host and test Apache configuration:

sudo a2ensite osticket.conf
sudo apache2ctl configtest

You should see: Syntax OK

Restart Apache

sudo systemctl restart apache2

Setup MySQL Database

Create a dedicated MySQL database and user account for osTicket with proper privileges.

5

Create Database & User

Set up MySQL authentication and database

Access MySQL Console

Log into MySQL with root account:

sudo mysql -u root -p

Enter your MySQL root password when prompted.

Create Database

Execute the following SQL commands to create the database:

CREATE DATABASE osticket;
CREATE USER 'osticket'@'localhost' IDENTIFIED BY 'securepassword123';
GRANT ALL PRIVILEGES ON osticket.* TO 'osticket'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Security Note: Replace 'securepassword123' with a strong, unique password. Use a mix of uppercase, lowercase, numbers, and special characters. Store this securely—you'll need it during installation.

Verify Database Connection

Test the osticket user can connect to the database:

mysql -u osticket -p osticket

If successful, type EXIT; to exit the MySQL console.

Deploy osTicket Application

Download and extract osTicket to the web root directory, then configure file permissions.

6

Download & Extract osTicket

Get the latest osTicket release from the official repository

Download osTicket

Navigate to the web root and download osTicket v1.16 (latest stable):

cd /var/www
sudo wget https://github.com/osTicket/osTicket/releases/download/v1.16.0/osTicket-v1.16.0.zip

Extract Archive

Extract the downloaded zip file:

sudo unzip osTicket-v1.16.0.zip
sudo rm osTicket-v1.16.0.zip

Set File Permissions

Configure proper file permissions for web server access:

sudo chown -R www-data:www-data /var/www/osticket
sudo chmod -R 755 /var/www/osticket/upload
sudo chmod 644 /var/www/osticket/upload/.htaccess

Make Configuration Writable

The installer needs to write the configuration file:

sudo chmod 777 /var/www/osticket/upload/include

After installation, you'll change this back to 755 for security.

7

Verify File Structure

Ensure osTicket files are correctly placed

List Directory Contents

Check that all osTicket files are present:

ls -la /var/www/osticket/upload/

You should see: index.php, include/, api/, admin/, client/ directories and other osTicket files.

Complete Installation Wizard

Access the web-based installer and complete the osTicket setup process. This wizard will configure the application and create initial settings.

8

Access Installation Wizard

Open osTicket in your browser and begin setup

Navigate to Installation Page

Open your web browser and go to:

http://osticket.local/setup/install.php

Replace osticket.local with your server's IP or domain name if not using local network.

Step 1: System Check

The installer will display a system compatibility check. Ensure all required items are marked as "OK" (green):

  • PHP Version 7.2+ (should show 7.4.x)
  • cURL Support
  • MySQL Support
  • IMAP Support
  • All recommended extensions

If any are marked as failed, go back and install missing PHP modules.

Step 2: Database Configuration

Enter the database credentials you created earlier:

  • MySQL Hostname: localhost
  • MySQL Database: osticket
  • MySQL Username: osticket
  • MySQL Password: [the password you set]
Checkpoint: After database configuration, you may see a message about downloading additional plugins. The installer will do this automatically. Be patient—this can take 1-2 minutes.
9

Configure osTicket

Set admin account, helpdesk info, and email settings

Step 3: osTicket Configuration

Fill in the basic configuration details:

  • Helpdesk Name: Your Organization Helpdesk
  • Helpdesk URL: http://osticket.local
  • Default Email: support@yourdomain.com
  • Timezone: Your desired timezone (e.g., UTC, EST, PST)

Step 4: Admin Account

Create the administrator account for osTicket:

  • First Name: Admin
  • Last Name: User
  • Email Address: admin@yourdomain.com
  • Username: admin (or your preferred username)
  • Password: Create a strong password (15+ characters)
Security: The admin password must be strong. Use a mix of uppercase, lowercase, numbers, and symbols. Store it securely and change it after first login.

Verify Installation & Security

Complete post-installation tasks to secure your osTicket instance and confirm everything is working correctly.

10

Post-Installation Security

Lock down file permissions and remove installation files

Secure Setup Directory

The setup/ directory should be removed or secured after installation:

sudo rm -rf /var/www/osticket/setup/

This prevents unauthorized access to the setup wizard.

Restrict Configuration Directory

Make the include/ directory read-only for security:

sudo chmod 755 /var/www/osticket/upload/include

Set Correct Ownership

Ensure all files are owned by the web server user:

sudo chown -R www-data:www-data /var/www/osticket
Note: In some setups, you may need to modify the include/ost-config.php file permissions. If you receive permission errors later, you can temporarily set it to 644, make changes, then reset to 644.
11

Verify Installation

Test both user and admin interfaces

Access Admin Panel

Log in to the admin interface:

http://osticket.local/admin/

Use the admin credentials you created during installation.

Access Client Portal

Verify the client-facing portal is accessible:

http://osticket.local/

You should see the client ticket submission form.

Test Ticket Creation

Create a test ticket to verify the system works:

  • Fill in the client portal form with test data
  • Submit the ticket
  • Note the ticket number provided
  • Log into the admin panel and verify the ticket appears
  • Verify email notifications (check spam folder if needed)

Common Issues & Troubleshooting

Issue: "No such file or directory" when accessing osticket.local

Solution: This typically means the virtual host is not pointing to the correct directory. Verify the DocumentRoot in your osticket.conf file is set to /var/www/osticket/upload. Also check that your local hosts file (or DNS) has an entry for osticket.local pointing to your server's IP.

Issue: "Database connection failed" during wizard

Solution: Double-check the database credentials you entered match exactly what you created: mysql -u osticket -p to verify the user can connect. Check MySQL is running with: sudo systemctl status mysql

Issue: PHP modules showing as missing in system check

Solution: Ensure you're using the correct PHP version. Run: php -v and verify it matches the modules you installed. If multiple PHP versions exist, update your Apache configuration to use the correct version.
